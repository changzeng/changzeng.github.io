EM算法

Veröffentlicht am 2018-02-06 | in machine learning

EM算法可以概括为通过已知当前步的参数推测下一步的参数，经过多次迭代后参数趋于收敛。

EM算法可用于包含隐变量的贝叶斯网络的求解，但不限于求解包含隐变量的贝叶斯网络。k-means就是一个经典的EM算法。

EM求解包含隐变量的贝叶斯网络可概括为先根据前一步的隐变量估计根部隐变量的值，这一步称为E步，M步则为根据E步所得隐变量估计叶子部分隐变量的值。

具体实例

假设有3枚硬币，分别记做A，B，C。这些硬币正面出现的概率分别是π,p, 和 q。进行如下掷硬币试验：先掷硬币A，根据其结果选出硬币B或硬币C，正面选硬币B，反面选硬币C；然后掷选出的硬币，掷硬币的结果，出现正面记做1，出现反面记做0；独立地重复n次试验（这里， n=10），观测结果如下：

1, 1, 0, 1, 0, 0, 1, 0, 1, 1

假设只能观测到掷硬币的结果，不能观察到掷硬币的过程，问如何估计三枚硬币正面出现的概率，即三硬币模型的参数（π,p,q ）。

上述问题可看做如下的一个贝叶斯网络。

上述硬币问题的似然函数为：

$$P(Y|\varTheta) = \prod{[\pi p^{y_j}(1-p)^{1-y_j} + (1-\pi)q^{y_j}(1-q)^{1-y_j}]}$$

考虑求模型参数Θ=(π,p,q)的极大似然估计，即

$$\varTheta = argmax_\varTheta\log{P(Y|\varTheta)}$$

  • E步：计算在模型参数$π_i,p_i,q_i$下观测数据$y_j$来自掷硬币B的概率

    $$\mu_{j}{(i+1)} = \frac{\pi^{(i)}(p^{(i)})^{y_i}(1-p^{(i)})^{1-y_i}}{\pi^{(i)}(p^{(i)})^{y_i}(1-p^{(i)})^{1-y_i} + (1-\pi^{(i)})(q^{(i)})^{y_i}(1-q^{(i)})^{1-y_i}}$$

  • M步：计算模型参数的新估计值：

    $$\pi^{(i+1)} = \frac{1}{n}\sum_{j=1}^{n}\mu_j^{(i+1)}$$

    $p^{(i+1)} = \frac{\sum_{j=1}^{n}\mu_j^{(i+1)}yj}{\sum{j=1}^{n}\mu_j^{i+1}}$(条件概率，在抽到硬币B的条件下翻到正面的概率）

    $q^{(i+1)} = \frac{\sum_{j=1}^{n}(1-\mu_j^{(i+1)})yj}{\sum{j=1}^{n}(1-\mu_j^{i+1})}$（条件概率，在抽到硬币C的条件下翻到正面的概率）

概率图模型中有向与无向的区别

Veröffentlicht am 2018-02-06 | in machine learning

共同点

将复杂的分布转化为多个因子的乘积。

不同点

有向图模型因子是概率分布，无需全局归一。

无向图模型因子是势函数，需要全局归一。

优缺点

无向图模型中势函数设计不受概率分布约束

设计灵活，但全局归一代价高

有向图模型无需全局归一、训练相对高效

推荐系统实战(二)之隐语义模型

Veröffentlicht am 2018-02-05 | in 推荐系统

LFM(Latent Factor Model)

总的来说LFM模型通过对用户历史数据的学习得到一系列的隐向量。而这些隐向量就是我们进行推荐的关键。最基础的LFM模型就是通过用户隐向量和物品隐向量的乘积得到用户对新向量的喜爱程度。

LFM模型是一种矩阵分解模型，它的优化目标就是使得得到的分解矩阵与原矩阵中非零元素的差值最小，当然为了避免过拟合，还需要在损失函数中加入L2正则化因子。

推荐系统实战(一)之协同过滤

Veröffentlicht am 2018-02-03 | in 推荐系统

历史点击数据分析

假定A，B两篇文章被点击的情况如下：

文章A 非文章A 总计
文章B 15 5 20
非文章B 75 5 80
总计 90 10 100

那么基于这样一个点击数据，我们就能得到加入一个用户点击了文章A后继续点击文章B的概率，即：

$P(A|B) = 15/20 = 0.75$

$P(B|A) = 15/90 = 0.167$

这样做的好处非常的直接，并不需要用户太多的历史点击数据，我们就能够进行推荐。但是缺点如下：

  1. 餐厅-客户个性不能被体现。
  2. 预定少的餐厅容易被剔除在外。
  3. 马太效应：热门餐厅获得推荐的机会更多。

Item Based协同过滤

下表中每一格代表两篇文章都被点击的次数，最末一行代表用户的偏好。

A B C D E F G
A 5 3 4 4 2 2 1
B 3 3 3 2 1 1 0
C 4 3 4 3 1 2 0
D 4 2 3 4 2 2 1
E 2 1 1 2 2 1 1
F 2 1 2 2 1 2 0
G 1 0 0 1 1 0 1
User 2.0 0.0 0.0 4.0 4.5 0.0 5.0

我们只要将用户偏好与点击矩阵相乘就能得到用户对于每一个物品的偏好度，这样做的原理其实本质上来说还是一个未正则化的余弦相似度的比较。

User Based协同过滤

基于用户的协同过滤需要事先计算出两两用户之间的相似度。然后根据不同用户对于同一物品的点击来计算用户对于某一物品的喜爱程度。

从一个实例来谈hadoop的mapreduce过程

Veröffentlicht am 2018-02-01 | in hadoop

假定现在我们有一个20G的文件，但是我们的单机内存只有1G，这个20G的文件每一行都是一个查询的字符串。现在我们的任务就是，如何计算每一个查询字符串出现的次数，并取得前k个。

这是一个典型的数据处理场景，其实对于海量的数据来说，不管我们机器的内存是200G还是300G都是一个非常小的容量。所以我们上述的问题是非常具有实际意义的。

下面给出问题的解法：

  1. 将文件切片。具体的做法是我们新建若干个文件，然后我们取查询字符串的哈希值模文件数目，就能得到该字符串所处的文件。这样一种操作所能带来的好处就是同样一个字符串一定能被分在同一个文件中，而总的文件数目又不会太大，而且每个文件的大小又能恰好被我们的机器所处理。
  2. 因为在我们对文件切片后，文件中的记录是无序的，所以接下来的步骤就是对每一个切分后的文件排序，这样我们就能把具有相同键值的字符串放到一起。
  3. 对相同的字符串进行合并，并对合并后的结果根据频率做排序。
  4. 由于我们需要频率最高的前k个，所以我们只需要取每个处理过后的文件的前k个作比较。我们在这里需要用到一个小顶堆保存取得的前k个，每次向堆中加入一条记录后去掉堆顶，所以我们总能保证堆中的元素始终是前k个。
  5. 最后，由于这是一个堆的结构，里面的元素是无序的，所以我们对这些元素进行排序后就能得到最终的结果。

其实在上述的过程中已经包含了mapreduce的全部思想，mapreduce的流程图如下所示。

左下黄色部分对应文件的切片，之所以切片是为了保证map的并行。map将文件映射为键值对，并为每一个键分配一个shuffle，为键值对分配shuffle的过程其实就是取键的hash值然后模一个数得到被分配到的shuflle序号。在shuffle中，我们首先需要对键值对排序，然后合并相同的键值对，然后将得到的结果送入reducer。

短文本分类

Veröffentlicht am 2018-01-30 | in nlp

短文本分类因为特征少，无法得到好的效果。另外，文本短提供的信息有限，需要的样本大，无法通过人工标注来做（减少人工）。

如何扩展特征？

长文本分类方法。84%
尝试用Word2vec对tfidf大的词汇进行扩展。 不可用。
尝试用lda来扩展特征。 86%. —— 因为lda扩展的特征有限。一般一条微博只能扩到1到2个特征。
尝试使用bigram来做特征扩展，效果最好。基于libshorttext。 95% 以上。 —— 就是textgrocery。

如何获取训练文本？

1.通过纯度较高的微博分类账户作为初始训练数据。
2.根据已有数据建立高招回分类器来扩展正例。
3.对已有正例的特征做皮尔逊相关系数排序。用排名较高的词汇进行solr搜索来获取正例。

更好的拟合现有数据？ 将训练集的badcase减少

对badcase中svm排序最高来扩展正例。计算badcase的相关系数，特征使用词+bigram。 选取正例加入到训练集。

SVM性能评测

Veröffentlicht am 2018-01-26 | in machine learning
  1. 在样本量比较少的时候，容易抓住数据和特征之间的非线性关系（相比线性分类方法如logistic regression，或者linear SVM）。但是，在样本量比较多的时候，线性分类方法的劣势就要小了很多，例如可以通过手工拆分/离散化特征来模拟非线性关系。这时SVM优势就不明显了。而且有个经验就是，在数据量大的时候，一些看起来很粗暴无脑的方法反而有“令人惊奇的好效果”。
  2. 计算复杂度高。主流的算法是O(n^2)的，这样对大规模数据就显得很无力了。不仅如此，由于其存在两个对结果影响相当大的超参数（如果用RBF核，是核函数的参数gamma以及惩罚项C），这两个超参数无法通过概率方法进行计算，只能通过穷举试验来求出，计算时间要远高于不少类似的非线性分类器。虽然说也还可以优化，但是与其优化它，不如去优化一些理论更鲁棒的方法（比如GP classification），或者直接就用计算时间较低的暴力非线性方法（如random forest，这是O(nlogn)的方法）
