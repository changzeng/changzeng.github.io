感知机、线性回归、逻辑回归

Veröffentlicht am 2018-01-14 | in machine learning

感知机、线性回归、逻辑回归是三中和容易混淆的模型，三者的主要区别在于激活函数的选择。

对于感知机来说，它的激活函数为：

$$h(a)=\begin{cases}1, a\geq 0 \0, a<0 　　 \end{cases}$$

对于线性回归模型来说，它的激活函数为：

$$h(a)=a$$

对于逻辑回归模型来说，它的激活函数则为:

$$h(a) = \frac{1}{1 + e^{-a}}$$

比较

感知机处理的数据必须是线性可分的，对于线性不可分的数据，感知机将显得十分的鸡肋。

LR使得最终结果有了概率解释的能力（将结果限制在0-1之间），sigmoid为平滑函数，能够得到更好的分类结果，而step function为分段函数，对于分类的结果处理比较粗糙，非0即1，而不是返回一个分类的概率。

梯度下降、随机梯度下降、批梯度下降

Veröffentlicht am 2018-01-14 | in deep learning

介绍

最近，看了斯坦福大学讲的梯度下降算法的视频，对其中的批量梯度下降算法(batch gradient descent algorithm，BGD)和随机梯度下降算法(Stochastic gradient descent algorithm，SGD)的区别还是不太理解。在网上查了一下，给出自己的理解：

我在这里所说的并不是官方的说法，而是对网上一些资料的总结。在有的资料中，批梯度下降和随机梯度下降指的是同一个东西，在有的资料中则区别对待。在这里我们假定这二者是有区别的。

这三者在原理上并没有区别，都是以梯度的方式迭代。但是他们三者最主要的区别就在于所使用样本的大小。

梯度下降使用的样本是总体样本。

批梯度下降使用的是一部份样本。

随机梯度下降使用的是单个样本。

总的来说，批梯度下降和随机梯度下降适合于低精度的任务，而梯度下降算法适合用于高精度的任务。

如果接受一个比较低的精度（应用问题中往往不要求高精度），那么由于SGD每次只利用一个或部分样本的梯度做更新，所以前期迭代较快，导致前期一段时间内比梯度下降算法下降得多。但是由于原始的SGD算法在目标函数强凸的情况下依旧无法做到线性收敛，所以当执行的时候足够长的时候SGD的精度会被梯度下降算法赶超，因为梯度下降算法可以在目标函数强凸的时候有线性收敛速度。

优化算法

常见的优化算法有

  • SGD
  • Momentum
  • Nesterov momentum
  • AdaGrad
  • RMSProp
  • ADAM
  • Second Order optimization methods

他们的性能差别可以通过下面的gif生动的表示。

SGD

从图中可以看出SGD容易陷入局部鞍点震荡，收敛速度较慢。

Momentum

Momentum可以使SGD不至于陷入局部鞍点震荡，同时起到一定加速作用。 Momentum最开始有可能会偏离较远(overshooting the target)，但是通常会慢慢矫正回来。

Nesterov momentum

基本思路是每次不在x位置求dx，而是在x+mu*v处更新dx，然后在用动量公式进行计算相当于每次先到动量的位置，然后求梯度更新

$$vt = \mu v{t-1} - \varepsilon \bigtriangledown f(\theta {t-1}+\mu v{t-1})$$

其中$\thetat = \theta{t-1}+v_t$

AdaGrad

使用每个变量的历史梯度值累加作为更新的分母，起到平衡不同变量梯度数值差异过大的问题

RMSProp

在AdaGrad基础上加入了decay factor，防止历史梯度求和过大

ADAM

Second Order optimization methods

作用/目的

跳表作为一种数据结构通常用于取代平衡树。跳表具有和红黑树相近的效率但是却在实现上更为简单，所以是红黑树的一种替代品。并且跳表能够实现高效的区间查询，redis的索引使用的就是跳表。

介绍

搜索算法

通常，我们从当前链表的最大等级的header开始搜索，如果同一等级节点的key值小于搜索值，则搜索相同等级的后续节点，否则，进入到下一个等级节点继续搜索。直到搜索到相应的值或已到最低等级而后续节点的值又大于当前搜索值（表明搜索已失败）为止。

插入/删除

插入和删除节点只需要在搜索的基础上再进行简单的插入和删除操作，只是需要注意两个操作当中前向指针关系的处理，以及增加和减少链表等级后及时更新当前最大等级的值。插入的过程可见如下示意图：

牛顿法和梯度下降法的比较

Veröffentlicht am 2018-01-14 | in machine learning

牛顿法和梯度下降法都是求解最优化问题时常用的算法，但是他们的区别在哪呢？

  1. 牛顿法起始点不能离局部极小点太远，否则很可能不会收敛。(考虑到二阶拟合应该很容易想象)，所以实际操作中会先使用别的方法，比如梯度下降法，使更新的点离最优点比较近，再开始用牛顿法。

  2. 牛顿法每次需要更新一个二阶矩阵，当维数增加的时候是非常耗内存的，所以实际使用是会用拟牛顿法。

  3. 梯度下降法在非常靠近最优点时会有震荡，就是说明明离的很近了，却很难到达，因为线性的逼近非常容易一个方向过去就过了最优点(因为只能是负梯度方向)。但牛顿法因为是二次收敛就很容易到达了。

条件随机场

Veröffentlicht am 2018-01-13 | in machine learning

前言

$p(y|x)$是crf的核心，这个公式的含义就是我们给定一个字符串，然后求解任意给定的标注序列的概率。比如在分词应用中，我们需要标注的label有$B,M,E,S$，那么假定我们给定一个字符串$我爱北京天安门$，那么crf就需要计算出任意给定的标注序列的概率，比如我们现在给定的标注序列是$B,E,B,E,B,M,E$，所有$p(y|x)$所需要计算的概率就是$p(B,E,B,E,B,M,E|我爱北京天安门)$。

数学形式

那么在数学上我们又该怎样表示这样一种关系呢？首先我们在任意位置抽取特征，然后训练得到这些特征的权值，最后通过这些训练的权值我们就能用来对新的数据做估计。

给定序列$X=(X_1,X_2,…,X_n)$，$Y=(Y_1,Y_2,…,Y_n)$，那么：

$$p(Y|X) = \frac{1}{Z(X)} exp(\sum_{i,k}\lambda_ktk(y{i-1},yi,x,i) + \sum{i,l}\mu_ls_l(y_i,x,i))$$

其中：

$$Z(X) = \sumy exp(\sum{i,k} \lambda_ktk(y{i-1,yi,x,i}) + \sum{i,l}\mu_ls_l(y_i,x,i))$$

在上面的公式中$t_k$和$\lambda_k$是二元特征及其对应的特征值，$s_l$和$\mu_l$是一元特征及其对应的特征值。所以crf的条件概率就能通过所有特征的权值的和来计算。

损失函数

L-BFGS属于无约束最小化算法，该算法在一些能用上批处理优化的ML问题中特别受欢迎。对于更大的数据集，则常用SGD方法，因为SGD只需要很少的迭代次数就能达到收敛。

给定训练样本${(x^m,y^m)}, m = {1,…,M}$，则CRF模型的对数似然函数为：

$$L(\lambda) = \sum_m log(P(y^m|x^m,\lambda)) = \sum_m(\sum_j \lambda_j f_j(x^m,y^m) - logZ(x^m))$$

在这里$f_j$和$\lambda_j$是特征及其所对应的特征值。我们的目标就是计算$\lambda$。

LDA与Gibbs Sampling

Veröffentlicht am 2018-01-13 | in machine learning

主题模型的应用十分广泛，LDA是其中最基础的一种。但是这个模型对于一般人来说十分的神秘，因为其中包含了很多的数学知识。初识LDA的时候我虽然把各种资料看了一遍又一遍，但是任然无法理解其中的奥秘。但是当我理解之后再回过头来看这些资料发现他们讲的确实详尽而有基础，但是它们都没有命中要害，而是把过多的部分放在了不必要的数学公式上了。当然，数学公式是理解模型的关键，但是如果一开始就从数学公式开始长篇大论不免会让不少人丧失了解的兴趣，而且也会让人觉得晦涩难懂。

LDA的目的就是为了计算每一篇文章的主题分布，以及每一主题的词分布。LDA模型分为在线模型和非在线模型，也就是说，非在线模型的主题分布是一次性计算，而在在线模型中一篇文章的主题分布可以通过已经训练的模型预估。

LDA的核心莫过于吉布斯采样，前面说过主题模型的作用就是要确定每一篇文章的主题分布，主题分布即文档中每一主题的出现频率。所以吉布斯采样的核心就是要为文档中的每一个词确定一个主题。那么这个主题如何确定呢？首先我们通过公式计算每一主题的概率，然后在得到的概率分布中随机选择一个。对整个语料库多次重复上述的采样过程，我们就能得到一个确定的主题分布。这就是吉布斯采样。

那么现在的问题就是，如何计算语料库中任意一词所属主题的概率以及为何主题分布会收敛。

先来说说收敛的问题，熟悉page rank的朋友一看到上面这个过程应该就是似曾相识，在page rank中，我们不停的遍历每一个网页并同时为其计算重要度，若干次迭代后网页的重要度便会收敛。其实我所说的page rank和吉布斯采样本质上来说并没有什么区别，都是从一个状态转移到另一个状态。然后最后状态收敛。熟悉马儿可夫链的同学一看到状态转移和收敛就很兴奋了，这不就是一个马尔科夫随机过程吗。对的，没错确实是这样。所以这样一种方法又被叫做蒙特卡罗-马尔科夫随机采样法，这种算法是一定会收敛的，但是证明很复杂也不要求要掌握，所以知道就行。

关于计算语料中任意词所属主题的概率问题。我想大部分的博客都已讲烂，我也不想赘述。但是我想说明的一点就是什么是共轭先验分布。共轭先验分布就是形式相同的分布，也就是说他们的概率都是用同一个公式计算的。

我想，明白了这个，现在你只需要稍微花一点时间，LDA便能信手拈来了。
