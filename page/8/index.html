条件随机场

Veröffentlicht am 2018-01-13 | in machine learning

前言

$p(y|x)$是crf的核心，这个公式的含义就是我们给定一个字符串，然后求解任意给定的标注序列的概率。比如在分词应用中，我们需要标注的label有$B,M,E,S$，那么假定我们给定一个字符串$我爱北京天安门$，那么crf就需要计算出任意给定的标注序列的概率，比如我们现在给定的标注序列是$B,E,B,E,B,M,E$，所有$p(y|x)$所需要计算的概率就是$p(B,E,B,E,B,M,E|我爱北京天安门)$。

数学形式

那么在数学上我们又该怎样表示这样一种关系呢？首先我们在任意位置抽取特征，然后训练得到这些特征的权值，最后通过这些训练的权值我们就能用来对新的数据做估计。

给定序列$X=(X_1,X_2,…,X_n)$，$Y=(Y_1,Y_2,…,Y_n)$，那么：

$$p(Y|X) = \frac{1}{Z(X)} exp(\sum_{i,k}\lambda_ktk(y{i-1},yi,x,i) + \sum{i,l}\mu_ls_l(y_i,x,i))$$

其中：

$$Z(X) = \sumy exp(\sum{i,k} \lambda_ktk(y{i-1,yi,x,i}) + \sum{i,l}\mu_ls_l(y_i,x,i))$$

在上面的公式中$t_k$和$\lambda_k$是二元特征及其对应的特征值，$s_l$和$\mu_l$是一元特征及其对应的特征值。所以crf的条件概率就能通过所有特征的权值的和来计算。

损失函数

L-BFGS属于无约束最小化算法，该算法在一些能用上批处理优化的ML问题中特别受欢迎。对于更大的数据集，则常用SGD方法，因为SGD只需要很少的迭代次数就能达到收敛。

给定训练样本${(x^m,y^m)}, m = {1,…,M}$，则CRF模型的对数似然函数为：

$$L(\lambda) = \sum_m log(P(y^m|x^m,\lambda)) = \sum_m(\sum_j \lambda_j f_j(x^m,y^m) - logZ(x^m))$$

在这里$f_j$和$\lambda_j$是特征及其所对应的特征值。我们的目标就是计算$\lambda$。

LDA与Gibbs Sampling

Veröffentlicht am 2018-01-13 | in machine learning

主题模型的应用十分广泛，LDA是其中最基础的一种。但是这个模型对于一般人来说十分的神秘，因为其中包含了很多的数学知识。初识LDA的时候我虽然把各种资料看了一遍又一遍，但是任然无法理解其中的奥秘。但是当我理解之后再回过头来看这些资料发现他们讲的确实详尽而有基础，但是它们都没有命中要害，而是把过多的部分放在了不必要的数学公式上了。当然，数学公式是理解模型的关键，但是如果一开始就从数学公式开始长篇大论不免会让不少人丧失了解的兴趣，而且也会让人觉得晦涩难懂。

LDA的目的就是为了计算每一篇文章的主题分布，以及每一主题的词分布。LDA模型分为在线模型和非在线模型，也就是说，非在线模型的主题分布是一次性计算，而在在线模型中一篇文章的主题分布可以通过已经训练的模型预估。

LDA的核心莫过于吉布斯采样，前面说过主题模型的作用就是要确定每一篇文章的主题分布，主题分布即文档中每一主题的出现频率。所以吉布斯采样的核心就是要为文档中的每一个词确定一个主题。那么这个主题如何确定呢？首先我们通过公式计算每一主题的概率，然后在得到的概率分布中随机选择一个。对整个语料库多次重复上述的采样过程，我们就能得到一个确定的主题分布。这就是吉布斯采样。

那么现在的问题就是，如何计算语料库中任意一词所属主题的概率以及为何主题分布会收敛。

先来说说收敛的问题，熟悉page rank的朋友一看到上面这个过程应该就是似曾相识，在page rank中，我们不停的遍历每一个网页并同时为其计算重要度，若干次迭代后网页的重要度便会收敛。其实我所说的page rank和吉布斯采样本质上来说并没有什么区别，都是从一个状态转移到另一个状态。然后最后状态收敛。熟悉马儿可夫链的同学一看到状态转移和收敛就很兴奋了，这不就是一个马尔科夫随机过程吗。对的，没错确实是这样。所以这样一种方法又被叫做蒙特卡罗-马尔科夫随机采样法，这种算法是一定会收敛的，但是证明很复杂也不要求要掌握，所以知道就行。

关于计算语料中任意词所属主题的概率问题。我想大部分的博客都已讲烂，我也不想赘述。但是我想说明的一点就是什么是共轭先验分布。共轭先验分布就是形式相同的分布，也就是说他们的概率都是用同一个公式计算的。

我想，明白了这个，现在你只需要稍微花一点时间，LDA便能信手拈来了。

Attention机制中权值的计算

Veröffentlicht am 2018-01-11 | in deep learning

我之前知道attention机制，但是对于如何计算attention的每一个隐藏层的权值这个东西，我是感到非常的迷茫，于是我翻阅各种资料终于找到了称心如意的答案。

话不多说，贴一个公式就都明白了。

$$e{i,j} = w^T tanh(Ws{i-1} + Vh_j + b)$$

再来一张图

现在就很直观了，对每一个位置计算一个权值，然后将所得到的所有的权值softmax之后就能得到每一位置的概率分布，然后将概率分布乘以对应位置的隐藏层状态再累加就能得到attention之后的encoder值了。

这种朴素的attention模型在序列长度较短的时候表现很好，但是在序列长度较长时计算量就会成倍增加，这是需要改进的地方。其实细想一下，这种attention并不是真正意义上的attention，因为他需要把序列遍历一遍之后再确定权值，这就好像人把一篇文章看完之后再去选择哪一个部分是最重要的，而人们直观上的注意力分布往往只会集中在一个点上，并不会把所有的要素都扫描一遍，所以这种attention比起叫注意力来说更像是从一堆数据中检索重要的部分。
